Resumen Ejecutivo

Conectar Envíos es una aplicación que conecta a transportistas freelance con clientes que necesitan enviar paquetes o realizar fletes. La plataforma permite a los transportistas ofrecer sus servicios de manera flexible, mientras los usuarios pueden obtener presupuestos y rastrear sus envíos en tiempo real. El objetivo de la app es optimizar el proceso de envíos (más que nada los de gran tamaño como mudanzas o compras grandes), ofreciendo una solución rápida y accesible, aprovechando la economía freelance.

Roles de Usuario

  • Cliente: Persona o empresa que necesita enviar un paquete o solicitar un flete.
  • Transportista: Freelancer que ofrece sus servicios de transporte de paquetes o fletes a través de la plataforma.
  • Administrador: Usuario encargado de gestionar el sistema, moderar contenido avanzado, gestionar cuentas, supervisar y administrar a los moderadores.
  • Moderador: Usuario con privilegios para supervisar la actividad dentro de la plataforma, incluyendo la moderación de reseñas, la revisión de disputas menores, y la vigilancia del cumplimiento de las políticas de la app.

Flujo de la Aplicación

  1. Registro e Inicio de Sesión: Los usuarios (clientes y transportistas) se registran con su correo electrónico o redes sociales.
  2. Publicación de Pedido: El cliente publica un pedido de envío, describiendo el paquete y el destino.
  3. Ofertas de Transportistas: Los transportistas ven los pedidos disponibles y envían ofertas con sus tarifas.
  4. Selección de Transportista: El cliente selecciona al transportista según el precio, la reputación y la disponibilidad.
  5. Seguimiento: El cliente rastrea el envío en tiempo real hasta que llegue a su destino.
  6. Calificación: Al finalizar el servicio, ambas partes se califican mutuamente para fomentar la confianza en la plataforma.

Funcionalidades Clave

  • Registro de usuarios: Registro de clientes y transportistas, con opciones de verificación de identidad.
  • Creación y gestión de pedidos: Los clientes pueden crear pedidos de flete con descripción, fotos y detalles.
  • Ofertas personalizadas: Los transportistas pueden enviar ofertas a los pedidos que les interesen.
  • Rastreo en tiempo real: Los clientes pueden seguir sus paquetes mediante geolocalización.
  • Calificaciones y reseñas: Sistema de calificaciones para mejorar la calidad de los servicios.
  • Notificaciones: Notificaciones en tiempo real para cambios de estado en los envíos.

Interfaz de Usuario (UI)

La interfaz será intuitiva y moderna, con un diseño minimalista para facilitar la navegación tanto en dispositivos móviles como en computadoras. Los clientes tendrán un panel para gestionar sus pedidos y rastrearlos, mientras que los transportistas podrán acceder a una vista de ofertas disponibles, con herramientas para aceptar trabajos y optimizar sus rutas.

Tecnología y Desarrollo

  • Frontend: Se utilizarán tecnologías como HTML, CSS y JavaScript (React o Vue.js) para crear una interfaz de usuario dinámica y responsiva.
  • Backend: Para el servidor y lógica de negocio, se usará Node.js o Django (Python), junto con una base de datos como PostgreSQL o MongoDB.
  • Geolocalización: Integración de servicios de mapas como Google Maps API para el rastreo en tiempo real.
  • Notificaciones Push: Implementación de notificaciones push para mantener a los usuarios informados sobre sus envíos.
  • API REST: El backend expondrá una API para gestionar la interacción entre la app móvil y el servidor.

Seguridad

  • Autenticación segura: Implementación de OAuth 2.0 para el inicio de sesión con verificación en dos pasos (2FA).
  • Encriptación: Todos los datos sensibles se transmitirán de forma encriptada utilizando HTTPS (SSL/TLS).
  • Roles y permisos: Los transportistas, clientes y administradores tendrán roles claramente definidos para evitar accesos no autorizados.
  • Protección contra ataques: Se integrarán medidas contra ataques comunes como SQL Injection y Cross-Site Scripting (XSS).

Competencia

En Argentina, los principales competidores en el mercado de envíos son UberFlash, Mercado Envíos y Cabify Envíos. Sin embargo, estas plataformas suelen centrarse en envíos pequeños y estandarizados, como paquetes que caben en un baúl. No ofrecen soluciones adecuadas para transportes de mayor tamaño, como mudanzas o el traslado de artículos voluminosos. Conectar Envíos llena este vacío, conectando a transportistas con vehículos más grandes y clientes que requieren envíos de mayor envergadura.

Monetización de la App

La aplicación Conectar Envíos busca generar ingresos mediante un enfoque que combine comisiones por transacción y un plan de suscripción premium. A continuación, se detallan las principales estrategias de monetización:

  • Comisión por Transacción:
    • Se aplicará una pequeña comisión sobre cada transacción realizada a través de la plataforma. Esta comisión será cobrada al transportista al momento de recibir el pago por el servicio.
    • Modelo de Pago: El cliente realizará el pago a través de Mercado Pago o mediante tarjeta de crédito o débito. Una vez que el transportista complete el servicio, podrá retirar sus ganancias, y en ese momento se le aplicará la comisión correspondiente.
    • Incentivo para el Plan Premium: La comisión estará diseñada de manera que, aunque sea competitiva, motive a los transportistas frecuentes a considerar la opción del plan premium para evitar cargos a futuro.
  • Suscripción Premium:
    • Los transportistas tendrán la opción de suscribirse a un plan premium con un cargo fijo mensual o anual. Este plan brindará beneficios adicionales:
    • Sin comisiones al retirar dinero: A diferencia del modelo estándar, los transportistas premium no estarán sujetos a comisiones al retirar el dinero de sus cuentas.
    • Funciones exclusivas: Los transportistas premium podrán acceder a features especiales, como mayor visibilidad en la plataforma, acceso prioritario a pedidos de alta demanda, reportes avanzados de sus ganancias, y herramientas para la optimización de rutas (detalles a confirmar).
    • Mayor límite de retiro: Se les ofrecerá la opción de realizar retiros más grandes sin restricciones de frecuencia o monto.
    • Costo del Plan: El precio del plan será competitivo y diseñado para atraer a transportistas que realizan múltiples trabajos al mes, haciéndolo más rentable en comparación con el modelo de comisión por transacción.

Prototipos

Conectar Envíos Standard (App de Clientes)

Pantalla de Registro: Permite a los clientes registrarse en la plataforma utilizando correo electrónico. Próximamente se podrá registrar con redes sociales o un número de teléfono.

Pantalla de Inicio de Sesión: Proporciona a los clientes un acceso seguro mediante credenciales previamente registradas.

Página Principal: Ofrece una vista general de los servicios disponibles, destacando pedidos en curso (si existe uno vigente) y opciones para crear un nuevo envío.

Crear Envío: Permite a los clientes ingresar los detalles del paquete, el destino y preferencias adicionales antes de seleccionar un transportista.

Envío en Proceso: Muestra información actualizada sobre el estado del envío y permite al cliente realizar seguimiento en tiempo real.

Estado del Envío: Detalla el progreso del envío, incluyendo ubicación del paquete y estimación de llegada.

Conectar Envíos Transportistas (App de Transportistas)

La app de transportistas permite a los freelancers gestionar ofertas, aceptar pedidos, y optimizar sus rutas. Pronto se añadiran los prototipos del lado del transportista.

Casos de Uso Alternativos o Excepcionales

1. Falsificación de Identidad del Transportista

Descripción: Un transportista se registra con información falsa o usa la cuenta de otra persona para ofrecer servicios de envío.

Solución Propuesta: Verificaciones de identidad obligatorias a través de documentos oficiales o integración con sistemas de verificación en línea, y verificación de antecedentes.

2. Robo de Mercadería

Descripción: El transportista alega que le han robado la mercadería o, en su defecto, roba el envío y culpa a terceros.

Solución Propuesta: Sistema de seguros obligatorios para envíos de alto valor y rastreo GPS en tiempo real, junto con un sistema de revisión de incidentes y disputas.

3. Cobro en Efectivo para Evadir Comisión

Descripción: El cliente y el transportista acuerdan realizar el pago en efectivo fuera de la plataforma, evitando la comisión.

Solución Propuesta: Penalizaciones para transportistas que no respeten el sistema de cobros y un sistema de reporte automático basado en indicadores sospechosos como usar la app pero siempre cancelar el envio.

4. Transportista Cancelando Repetidamente

Descripción: El transportista cancela trabajos repetidamente, afectando la experiencia del cliente.

Solución Propuesta: Penalizaciones o bloqueo temporal a transportistas con cancelaciones excesivas sin justificación.

5. Mal uso de la Plataforma por el Cliente

Descripción: Un cliente pide servicios de flete para transportar mercadería ilegal o peligrosa.

Solución Propuesta: Términos y condiciones claros que prohíban el uso indebido, y un sistema de reporte para transportistas. Permitir a los transportistas denunciar cualquier pedido sospechoso directamente desde la plataforma, sin penalizaciones si deciden no aceptar el trabajo por motivos de seguridad o legales.

6. Fallas en el Sistema de Rastreo en Tiempo Real

Descripción: El sistema de rastreo de la plataforma deja de funcionar durante un envío importante.

Solución Propuesta: Notificaciones automáticas por errores en el sistema de rastreo y un plan de contingencia para contacto directo con el transportista.

7. Pérdida de Datos o Ataques Cibernéticos

Descripción: Un ciberataque compromete la seguridad de los datos de los usuarios, como información personal o financiera.

Solución Propuesta: Encriptación avanzada, autenticación multifactor (2FA) y copias de seguridad automáticas para prevenir la pérdida de datos.

8. Disputas sobre el Pago por Servicios No Completados

Descripción: Un cliente reporta que el envío no fue completado, pero el sistema ya ha liberado el pago al transportista.

Solución Propuesta: Sistema de pago en depósito o escrow, donde los fondos se liberan solo después de que el cliente confirme la finalización del servicio.

9. Interrupción del Servicio por Caídas del Servidor

Descripción: La plataforma experimenta una caída del servidor, afectando la experiencia de los usuarios.

Solución Propuesta: Sistema de alta disponibilidad con servidores redundantes y alertas a los usuarios si el servicio se interrumpe temporalmente.

10. Manipulación de Calificaciones o Reseñas

Descripción: Transportistas o clientes intentan manipular el sistema de calificaciones dejando reseñas falsas.

Solución Propuesta: Algoritmos que detecten patrones sospechosos y eliminación de reseñas fraudulentas.

11. Transporte de Carga Prohibida o Dañina

Descripción: Un cliente solicita el transporte de materiales prohibidos por ley o peligrosos.

Solución Propuesta: Definir una lista de materiales prohibidos y permitir a los transportistas rechazar envíos sospechosos sin penalización.

12. Cliente Reclamando Entrega No Realizada

Descripción: Un cliente reclama que el envío nunca llegó, pero el transportista dice que lo entregó.

Solución Propuesta: Pruebas de entrega obligatorias, como fotos con firma del receptor o un PIN que el cliente debe proporcionar al transportista.
