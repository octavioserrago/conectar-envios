Requerimientos del Proyecto

Requerimientos Funcionales

Los requerimientos funcionales del sistema Conectar Envíos son los siguientes:

  • Registro y autenticación de usuarios (clientes y transportistas).
  • Gestión de envíos, incluyendo la creación y seguimiento de pedidos.
  • Gestión de rutas y optimización para transportistas.
  • Notificaciones sobre el estado de los envíos.

Requerimientos No Funcionales

Los requerimientos no funcionales incluyen:

  • La aplicación debe ser accesible desde dispositivos móviles y escritorios.
  • La plataforma debe manejar múltiples usuarios simultáneamente.
  • Los tiempos de carga de la aplicación no deben exceder los 3 segundos.
  • La plataforma debe contar con un sistema de recuperación ante fallos.

Requerimientos Técnicos

Los requisitos técnicos para el desarrollo de Conectar Envíos son los siguientes:

  • Frontend: Se utilizarán tecnologías como HTML, CSS y JavaScript (React o Vue.js) para crear una interfaz de usuario dinámica y responsiva.
  • Backend: Para el servidor y lógica de negocio, se usará Node.js o Django (Python), junto con una base de datos como PostgreSQL o MongoDB.
  • Geolocalización: Integración de servicios de mapas como Google Maps API para el rastreo en tiempo real.
  • Notificaciones Push: Implementación de notificaciones push para mantener a los usuarios informados sobre sus envíos.
  • API REST: El backend expondrá una API para gestionar la interacción entre la app móvil y el servidor.

Requerimientos de Seguridad

La plataforma debe garantizar los siguientes aspectos de seguridad:

  • Cifrado de contraseñas de usuarios mediante algoritmos seguros.
  • Protección contra ataques de inyección SQL y cross-site scripting (XSS).
  • Autenticación de dos factores para usuarios con permisos especiales.
