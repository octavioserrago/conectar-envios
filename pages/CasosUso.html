Casos de Uso

Caso de Uso 1: Registro de Cliente

Descripción: El cliente debe poder registrarse en la plataforma Conectar Envíos utilizando un correo electrónico. En el futuro, se añadirá la opción de registrarse mediante redes sociales o un número de teléfono.

Solución: Se crea una pantalla de registro en la que el cliente ingresa su correo electrónico, seguido de una contraseña y confirmación. Un botón de "Registrar" valida los datos e inicia el proceso de creación de cuenta. Si la información es válida, el sistema envía un correo electrónico de confirmación y crea la cuenta del cliente.

Caso de Uso 2: Inicio de Sesión de Cliente

Descripción: El cliente debe poder iniciar sesión en la plataforma utilizando sus credenciales de correo electrónico y contraseña.

Solución: Se crea una pantalla de inicio de sesión donde el cliente ingresa su correo electrónico y contraseña. Si las credenciales coinciden con las almacenadas en el sistema, el cliente es redirigido a la página principal. Si las credenciales no son correctas, se muestra un mensaje de error.

Caso de Uso 3: Creación de Envío

Descripción: El cliente debe poder crear un envío, proporcionando detalles como la dirección de destino, el paquete, y preferencias adicionales antes de seleccionar un transportista.

Solución: Se crea una pantalla donde el cliente ingresa la información del envío, como la dirección de origen y destino, tipo de paquete, peso, y cualquier otra preferencia. El cliente puede seleccionar el transportista preferido, y después de validar los datos, el sistema crea el pedido y notifica al cliente sobre la confirmación del envío.

Caso de Uso 4: Seguimiento de Envío

Descripción: El cliente debe poder realizar un seguimiento de sus envíos en tiempo real, con información actualizada sobre el estado y ubicación del paquete.

Solución: Se crea una página de seguimiento donde el cliente puede ver una lista de sus envíos activos, con detalles sobre el estado y ubicación de cada paquete. El cliente puede ver el progreso de cada envío en tiempo real y recibir notificaciones sobre el estado del paquete hasta su entrega.

Caso de Uso 5: Cancelación de Envío

Descripción: El cliente debe poder cancelar un envío antes de que se haya procesado o entregado, de acuerdo con las políticas de la plataforma.

Solución: Se crea una opción de cancelación en la página de seguimiento de envío. El cliente puede cancelar el envío si aún no ha sido procesado o está en un estado inicial. Después de la cancelación, el sistema notifica al cliente sobre la acción y actualiza el estado del envío a "Cancelado".

Caso de Uso 6: Registro de Transportista

Descripción: Los transportistas deben poder registrarse en la plataforma para ofrecer sus servicios y gestionar sus pedidos. Se les solicitará información adicional para garantizar que cumplen con los requisitos legales y de seguridad.

Solución: Se crea un formulario de registro específico para los transportistas, donde deben ingresar información como nombre, vehículos disponibles, tarifas y adjuntar los siguientes documentos:

  • Antecedentes penales (aún sujeta a revisión).
  • Cédula de identidad.
  • VTV de los vehículos asociados.
  • Seguro de los vehículos al día.

Al completar el registro con la documentación requerida, los transportistas reciben una notificación de confirmación y acceden a su panel de control para gestionar pedidos y rutas.

Caso de Uso 7: Visualización de Rutas

Descripción: El transportista debe poder visualizar las rutas asignadas, con información detallada sobre los destinos y el tiempo estimado de llegada.

Solución: Se crea una página de rutas donde el transportista puede ver las rutas asignadas, con una lista detallada de los destinos y el tiempo estimado de llegada. Además, se muestra información de contacto y cualquier preferencia del cliente relacionada con cada envío.

Caso de Uso 8: Recepción de Paquete

Descripción: El transportista debe registrar la recepción de un paquete en su sistema, lo que permitirá a la plataforma confirmar que el paquete ha sido recogido.

Solución: Se implementa un sistema de escaneo de código de barras o QR en la app del transportista para registrar la recepción del paquete. Al escanear el código del paquete, el sistema marca el paquete como "Recibido" y notifica al cliente.

Caso de Uso 9: Entrega de Paquete

Descripción: El transportista debe registrar la entrega de un paquete en la plataforma, lo que permite actualizar el estado de entrega y notificar al cliente.

Solución: Se implementa un sistema similar al de la recepción del paquete, donde el transportista escanea el código del paquete al momento de la entrega. El sistema actualiza el estado del paquete a "Entregado" y notifica al cliente.

Caso de Uso 10: Solicitar Reprogramación de Entrega

Descripción: El cliente debe poder solicitar la reprogramación de una entrega si no puede recibir el paquete en la fecha originalmente programada.

Solución: El cliente puede ingresar a la plataforma y solicitar la reprogramación de la entrega, eligiendo una nueva fecha de entrega. Si la fecha es disponible en el sistema, la entrega se reprograma y el cliente recibe una notificación. Si no es posible, se le ofrecen otras fechas disponibles.

Caso de Uso 11: Producto Dañado Durante el Transporte

Descripción: En caso de que el producto enviado se dañe durante el transporte, el cliente y el transportista deben seguir un proceso para reportar y gestionar la situación, incluidas las responsabilidades y posibles compensaciones.

Solución: Si el cliente recibe un producto dañado, debe notificarlo inmediatamente a través de la plataforma, proporcionando fotos del daño y una descripción detallada del estado del paquete. La plataforma creará un informe de incidente que será revisado por el transportista y el equipo de soporte. El transportista tendrá la oportunidad de dar su versión de los hechos, y se investigará si hubo negligencia o si el daño ocurrió durante el transporte. Si es determinado que el transportista es responsable, la plataforma gestionará una compensación o reembolso al cliente, y podría aplicar sanciones al transportista si fuera necesario. Además, se podrá ofrecer al cliente un reemplazo o un envío gratuito de un nuevo paquete, según el acuerdo entre las partes.

Caso de Uso 12: No Hay Quien Reciba el Paquete

Descripción: En caso de que el transportista llegue a la dirección de envío y no haya nadie disponible para recibir el paquete, y el cliente no responda, se debe tomar una acción adecuada para resolver la situación.

Solución: Cuando el transportista llega a la dirección de destino pero no encuentra a nadie para recibir el paquete y el cliente no responde a los intentos de contacto, la plataforma le ofrecerá al transportista la opción de devolver el paquete a su origen, si es posible. El transportista realizará la devolución y se procesará el pago correspondiente al mismo, tal como si el servicio fuera completado. El cliente no recibirá ningún reembolso por este incidente, ya que la situación se considera un incumplimiento por parte del receptor (cliente). La plataforma notificará al cliente sobre el intento de entrega fallido y explicará las condiciones de este incidente, incluyendo la no devolución del dinero. Si el cliente desea reprogramar el envío, deberá realizar un nuevo pago por el servicio.

Caso de Uso 13: Modificación de Dirección de Entrega

Descripción: El cliente debe poder modificar la dirección de entrega de su paquete antes de que el transportista lo recoja.

Solución: El cliente podrá acceder a la opción de modificar la dirección de entrega desde la página de seguimiento, siempre y cuando el paquete aún no haya sido recogido por el transportista. La plataforma validará que la nueva dirección sea válida y notificará tanto al cliente como al transportista sobre el cambio.

Caso de Uso 14: Calificación y Comentarios del Transportista

Descripción: El cliente debe poder calificar al transportista después de cada entrega, y dejar comentarios para proporcionar retroalimentación sobre el servicio.

Solución: Tras la entrega, la plataforma enviará una solicitud al cliente para calificar al transportista en una escala de 1 a 5 estrellas, y permitirle dejar un comentario opcional. Esta información ayudará a mejorar los servicios y a clasificar a los transportistas. Si el transportista recibe calificaciones bajas con frecuencia, la plataforma puede tomar medidas para mejorar el servicio o incluso suspender al transportista.

Caso de Uso 15: Soporte de Emergencia para el Transportista

Descripción: Los transportistas deben tener acceso a un servicio de soporte de emergencia en caso de que enfrenten problemas durante la entrega, como accidentes, problemas con el vehículo, o cualquier situación que les impida completar la entrega.

Solución: La plataforma ofrecerá un botón de "Soporte de Emergencia" dentro de la app para los transportistas, que les permitirá ponerse en contacto con el equipo de soporte para recibir ayuda inmediata. El equipo de soporte podrá guiar al transportista a través de posibles soluciones o enviar asistencia si es necesario.

Caso de Uso 16: Cambio de Método de Pago

Descripción: El cliente debe poder cambiar el método de pago para el servicio de envío antes de que se procese el pago, utilizando Mercado Pago como plataforma de pagos.

Solución: En la sección de pago, el cliente podrá seleccionar entre diferentes métodos de pago ofrecidos por Mercado Pago, como tarjeta de crédito, débito, o saldo disponible. Si el cliente desea cambiar el método de pago antes de que se procese la transacción, podrá hacerlo sin afectar la programación del envío. La plataforma actualizará el método de pago seleccionado en Mercado Pago y notificará al cliente y al transportista sobre la actualización.

Caso de Uso 17: Validación de Documentos del Transportista

Descripción: Antes de aceptar un transportista en la plataforma, los documentos relevantes (como la cédula de identidad, los antecedentes penales, la VTV y el seguro) deben ser validados para asegurar que el transportista cumpla con los requisitos legales y de seguridad.

Solución: Durante el proceso de registro, los transportistas deberán subir imágenes de los documentos requeridos. La plataforma contará con un sistema de validación automática o un equipo de soporte que verificará la autenticidad de los documentos antes de aprobar al transportista. Si los documentos no son válidos o faltan, el transportista no podrá ofrecer sus servicios en la plataforma.

Caso de Uso 18: Verificación de Identidad en la Entrega

Descripción: Antes de entregar un paquete, el transportista debe verificar la identidad del receptor solicitando un DNI o una clave única, que el cliente puede visualizar en su app.

Solución: Se implementa un sistema en la app del transportista que, antes de confirmar la entrega, solicita al receptor ingresar un DNI o una clave única proporcionada por la plataforma. Esta clave es generada automáticamente al momento de crear el envío y es visible solo para el cliente en su app. El flujo sería el siguiente:

  • El transportista selecciona el envío que desea entregar en su app.
  • El sistema solicita al receptor ingresar el DNI o la clave única.
  • Si la información ingresada coincide con la almacenada en el sistema, se confirma la entrega y se actualiza el estado del paquete a "Entregado".
  • Si la información no es válida, el sistema muestra un mensaje de error y no permite completar la entrega.

Este proceso garantiza que el paquete sea recibido únicamente por el destinatario autorizado, mejorando la seguridad del servicio.

Caso de Uso 19: Notificación Proactiva de Retrasos

Descripción: El cliente debe recibir notificaciones automáticas si el transportista experimenta retrasos debido a tráfico, condiciones climáticas o problemas técnicos.

Solución: Se implementa un sistema de monitoreo en tiempo real que detecta retrasos en las rutas asignadas al transportista. Si el tiempo estimado de llegada cambia significativamente, la plataforma envía notificaciones al cliente indicando el motivo del retraso y un nuevo tiempo estimado de entrega.

Caso de Uso 20: Control de Entregas con Temperatura Regulada

Descripción: Permitir a los clientes solicitar servicios especiales para envíos que requieren temperatura controlada (como alimentos o productos farmacéuticos).

Solución: Se agrega una opción en la creación de envíos para especificar requisitos especiales como "Temperatura regulada". Los transportistas con vehículos equipados para este servicio serán asignados automáticamente. La plataforma supervisará el cumplimiento mediante sensores de temperatura conectados al sistema.

Nota: Además, se solicitará siempre a los transportistas que detallen el estado o último servicio técnico realizado al equipo de refrigeración, garantizando así que los envíos se realicen bajo condiciones óptimas.

Caso de Uso 21: Programa de Recompensas

Descripción: Ofrecer un sistema de recompensas para clientes frecuentes que utilicen la plataforma regularmente.

Solución: Se crea un programa de puntos donde cada envío genera puntos canjeables por descuentos en futuros envíos. El sistema llevará un registro de los puntos acumulados y notificará al cliente cuando tenga suficientes puntos para obtener un beneficio.

Conclusión

Los casos de uso y sus soluciones ofrecen una visión integral de las funcionalidades y flujos de la plataforma Conectar Envíos. Estos procesos están diseñados para ser intuitivos y eficientes, garantizando la satisfacción tanto de los clientes como de los transportistas.
