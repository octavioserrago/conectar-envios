Casos de Uso

Caso de Uso 1: Registro de Cliente

Descripción: El cliente debe poder registrarse en la plataforma Conectar Envíos utilizando un correo electrónico. En el futuro, se añadirá la opción de registrarse mediante redes sociales o un número de teléfono.

Solución: Se crea una pantalla de registro en la que el cliente ingresa su correo electrónico, seguido de una contraseña y confirmación. Un botón de "Registrar" valida los datos e inicia el proceso de creación de cuenta. Si la información es válida, el sistema envía un correo electrónico de confirmación y crea la cuenta del cliente.

Caso de Uso 2: Inicio de Sesión de Cliente

Descripción: El cliente debe poder iniciar sesión en la plataforma utilizando sus credenciales de correo electrónico y contraseña.

Solución: Se crea una pantalla de inicio de sesión donde el cliente ingresa su correo electrónico y contraseña. Si las credenciales coinciden con las almacenadas en el sistema, el cliente es redirigido a la página principal. Si las credenciales no son correctas, se muestra un mensaje de error.

Caso de Uso 3: Creación de Envío

Descripción: El cliente debe poder crear un envío, proporcionando detalles como la dirección de destino, el paquete, y preferencias adicionales antes de seleccionar un transportista.

Solución: Se crea una pantalla donde el cliente ingresa la información del envío, como la dirección de origen y destino, tipo de paquete, peso, y cualquier otra preferencia. El cliente puede seleccionar el transportista preferido, y después de validar los datos, el sistema crea el pedido y notifica al cliente sobre la confirmación del envío.

Caso de Uso 4: Seguimiento de Envío

Descripción: El cliente debe poder realizar un seguimiento de sus envíos en tiempo real, con información actualizada sobre el estado y ubicación del paquete.

Solución: Se crea una página de seguimiento donde el cliente puede ver una lista de sus envíos activos, con detalles sobre el estado y ubicación de cada paquete. El cliente puede ver el progreso de cada envío en tiempo real y recibir notificaciones sobre el estado del paquete hasta su entrega.

Caso de Uso 5: Cancelación de Envío

Descripción: El cliente debe poder cancelar un envío antes de que se haya procesado o entregado, de acuerdo con las políticas de la plataforma.

Solución: Se crea una opción de cancelación en la página de seguimiento de envío. El cliente puede cancelar el envío si aún no ha sido procesado o está en un estado inicial. Después de la cancelación, el sistema notifica al cliente sobre la acción y actualiza el estado del envío a "Cancelado".

Caso de Uso 6: Registro de Transportista

Descripción: Los transportistas deben poder registrarse en la plataforma para ofrecer sus servicios y gestionar sus pedidos. Se les solicitará información adicional para garantizar que cumplen con los requisitos legales y de seguridad.

Solución: Se crea un formulario de registro específico para los transportistas, donde deben ingresar información como nombre, vehículos disponibles, tarifas y adjuntar los siguientes documentos:

  • Antecedentes penales (aún sujeta a revisión).
  • Cédula de identidad.
  • VTV de los vehículos asociados.
  • Seguro de los vehículos al día.

Al completar el registro con la documentación requerida, los transportistas reciben una notificación de confirmación y acceden a su panel de control para gestionar pedidos y rutas.

Caso de Uso 7: Visualización de Rutas

Descripción: El transportista debe poder visualizar las rutas asignadas, con información detallada sobre los destinos y el tiempo estimado de llegada.

Solución: Se crea una página de rutas donde el transportista puede ver las rutas asignadas, con una lista detallada de los destinos y el tiempo estimado de llegada. Además, se muestra información de contacto y cualquier preferencia del cliente relacionada con cada envío.

Caso de Uso 8: Calificación de Servicio

Descripción: El cliente debe poder calificar el servicio del transportista al finalizar el envío, lo que permitirá mejorar la calidad del servicio.

Solución: Se crea una opción en la que el cliente puede dar una calificación al transportista después de recibir el paquete. El cliente puede elegir entre una escala de 1 a 5 estrellas y dejar comentarios adicionales. Esta retroalimentación se almacena y se utiliza para mejorar el servicio.

Conclusión

Los casos de uso y sus soluciones ofrecen una visión integral de las funcionalidades y flujos de la plataforma Conectar Envíos. Estos procesos están diseñados para ser intuitivos y eficientes, garantizando la satisfacción tanto de los clientes como de los transportistas.
